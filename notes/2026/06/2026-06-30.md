## First-Submit Diagnostics

### Implementation
- Added explicit manual-verification/CAPTCHA-style challenge classification to the create-page extractor.
- Added `generation_pre_submit` evidence so runs capture safe pre-click diagnostics before Create is pressed.
- Added `--confirm-submit` mode to fill a request, inspect pre-submit state, and stop before Create.
- Hardened submit behavior so visible manual verification, missing Create controls, or disabled Create controls block before any submit click.
- Wired configured post-login warmup into the create flow.
- Switched create-form field entry toward visible focused controls and keyboard typing, with normal locator fallbacks.
- Tuned typing cadence after local headed testing showed the first version was visibly too slow.
- Updated README, setup, architecture, create-box, manual-smoke, and observability docs for the new diagnostics and operating boundary.
